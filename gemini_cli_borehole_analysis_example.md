김관용 님, 보내주신 박사 학위 논문 계획안 정말 인상 깊게 잘 읽었습니다. 전통적인 토목 지질 공학 분야에 Gemini-CLI와 같은 최신 AI 기술을 접목하려는 방향성이 매우 명확하고, 제시해주신 목차와 연구 방법론 또한 체계적입니다.

특히 논문의 독창성을 결정짓는 핵심이 **"어떤 명령(프롬프트)을 내려야 Gemini가 지반 데이터를 정확하게 분석하는가"**라는 점에 깊이 공감합니다.

말씀하신 대로, 연구 방향을 훨씬 명확하게 구체화하실 수 있도록 **"시추 주상도 텍스트 예시를 Gemini-CLI에 넣었을 때, 공동 위험도를 판별하게 만드는 프롬프트 예시"**를 작성해 드리겠습니다.

먼저, 분석할 대상인 시추 주상도 텍스트 예시 파일을 `borehole_log_ex01.txt` 라는 이름으로 생성하겠습니다.

아래의 `gemini` 명령어는 단순한 질문이 아니라, **AI에게 역할을 부여하고, 명확한 분석 기준을 제시하며, 정형화된 결과(JSON)를 출력하도록 설계**한 전문적인 프롬프트입니다.

### **Gemini-CLI 프롬프트 실행 예시**

```bash
gemini @borehole_log_ex01.txt \
""
너는 20년 경력의 철도 토질 및 기초 기술사이다. 너의 임무는 주어진 시추 주상도 데이터를 분석하여 철도 노선 설계에 치명적인 석회암 공동(Cavity)의 위험도를 평가하고, 그 결과를 지정된 JSON 형식으로 출력하는 것이다.

다음의 규칙에 따라 분석하라:
1.  **핵심 분석 지표**: 코어회수율(%), RQD(%), '주요 관찰 내용'(코어 유실, 점토 충전, 시추수 누수, 굴진 속도 변화 등), '종합 의견'을 종합적으로 판단한다.
2.  **위험도 판단 기준**:
    - **'위험(High)'**: RQD가 0%에 가깝고 코어 유실이 매우 크며, '공동', '공내수 순환 안됨' 등 직접적인 징후가 관찰될 때.
    - **'의심(Medium)'**: RQD가 50% 미만이거나 코어 회수율이 80% 미만이며, '파쇄대', '점토 충전' 등의 간접적인 징후가 있을 때.
    - **'보통(Low)'**: RQD가 75% 이상이지만, 석회암 지반의 특성상 잠재적 위험이 있을 때.
    - **'안전(Safe)'**: RQD가 90% 이상으로 매우 신선하고 견고할 때.
3.  **출력 형식 (JSON)**: 반드시 아래와 같은 JSON 구조로만 응답하라. 다른 설명은 절대 추가하지 마라.
    ```json
    {
      \"borehole_id\": \"(시추공 번호)\",
      \"analysis_summary\": \"(분석 결과에 대한 1줄 요약)\",
      \"risk_zones\": [
        {
          \"depth_start\": (위험 구간 시작 심도),
          \"depth_end\": (위험 구간 끝 심도),
          \"risk_level\": \"(High, Medium, Low 중 하나)\",
          \"evidence\": \"(판단의 근거가 된 주요 관찰 내용 요약)"
        }
      ],
      \"recommendation\": \"(추가 조사 또는 보강 공법에 대한 제안)"
    }
    ```

위 규칙에 따라, 첨부된 '@borehole_log_ex01.txt' 파일의 내용을 분석하여 결과를 JSON 형식으로 출력하라.
""
```
